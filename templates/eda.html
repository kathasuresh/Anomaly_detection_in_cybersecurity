Exploratory Data Analysis

Visual insights and distributions of key attributes extracted during model exploration.

1. Class Balance (Attack vs Normal)

Insights: Out of 9,537 total sessions, {{ metadata.dataset.normal_count }} (55.3%) are classified as normal, while {{ metadata.dataset.attack_count }} (44.7%) represent cyber attacks/intrusions. This relatively balanced dataset ensures that our models learn features of both normal and attack behaviors without major class bias.

2. IP Reputation vs Intrusion

Insights: Normal sessions correlate strongly with higher IP reputation scores (median around 0.8), while detected attacks exhibit significantly lower IP reputation scores (median below 0.35). This indicates that IP reputation is a highly discriminative indicator of malicious activity.

3. Feature Correlation Matrix

Insights: - The correlation matrix shows a strong positive correlation between `failed_logins`, `login_attempts`, and the target label `attack_detected`. - Engineered features like `login_risk_score` (attempts × failed) show powerful correlation alignment with malicious sessions. - Categorical variables (such as protocol and browser types) exhibit low correlation values, suggesting their relationship is non-linear and better captured by decision trees or ensemble algorithms like Random Forest.

4. Packet Size & Session Duration Distributions

Insights: - Session Duration: Cyber attacks (red) are frequently clustered in very short session intervals (less than 15 minutes), likely indicating rapid automated scanning or credential stuffing tools. Normal sessions (blue) display a wider, more uniform duration distribution. - Network Packet Size: Intrusions display spike frequencies at specific packet size thresholds, corresponding to malicious payload signatures. Normal sessions exhibit a smoother, right-skewed distribution.

5. Receiver Operating Characteristic (ROC) Curve

Insights: The ROC curve illustrates the diagnostic ability of our classifiers. - Gradient Boosting and Random Forest both show outstanding diagnostic power, achieving Area Under the Curve (AUC) scores of **~0.95+**. - This high AUC indicates an extremely high true-positive rate with minimal false-positives across various decision thresholds.

6. SHAP Feature Contribution Summary

Insights: SHAP (SHapley Additive exPlanations) values decompose predictions to explain feature impact: - **High values of failed logins** (red dots) push the model heavily towards an attack prediction (positive SHAP). - **Low values of IP reputation score** (blue dots) push the model towards detecting an intrusion. - This explains not just *what* the model predicts, but *why* it predicts it, ensuring explainability (XAI) for cybersecurity audits.
